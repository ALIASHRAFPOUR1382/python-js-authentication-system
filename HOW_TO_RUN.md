# راهنمای اجرای پروژه

## مشکل 404 - File not found

اگر خطای 404 دریافت می‌کنید، احتمالاً یکی از این مشکلات است:

### 1. سرور در حال اجرا نیست

**راه حل:**
1. سرور را اجرا کنید:
   ```bash
   cd backend
   python server.py
   ```
   
   یا در Windows:
   ```bash
   start_server.bat
   ```

2. منتظر بمانید تا پیام زیر نمایش داده شود:
   ```
   Server running on http://localhost:8000
   ```

### 2. فایل را مستقیماً باز کرده‌اید

**مشکل:** اگر فایل `auth.html` را مستقیماً در مرورگر باز کنید (با دابل کلیک)، خطای 404 می‌گیرید.

**راه حل:** باید از طریق سرور باز کنید:
```
http://localhost:8000/auth.html
```

### 3. مسیر اشتباه

**راه حل:** مطمئن شوید که:
- سرور در حال اجرا است
- به آدرس `http://localhost:8000/auth.html` می‌روید
- نه `file:///C:/Users/.../auth.html`

## مراحل اجرا

### مرحله 1: اجرای سرور

```bash
cd backend
python server.py
```

### مرحله 2: باز کردن مرورگر

در مرورگر به آدرس زیر بروید:
```
http://localhost:8000/auth.html
```

یا برای صفحه اصلی:
```
http://localhost:8000
```

### مرحله 3: استفاده

- برای لاگین: روی "Login" کلیک کنید
- برای ثبت‌نام: روی "Sign up" کلیک کنید

## بررسی مشکلات

### بررسی 1: آیا سرور در حال اجرا است؟

در ترمینال باید پیام زیر را ببینید:
```
Server running on http://localhost:8000
Press Ctrl+C to stop the server
```

### بررسی 2: آیا فایل‌ها وجود دارند؟

مطمئن شوید که این فایل‌ها وجود دارند:
- `frontend/auth.html`
- `frontend/css/style.css`
- `frontend/js/auth.js`
- `frontend/js/auth-switch.js`

### بررسی 3: آیا پورت 8000 آزاد است؟

اگر پورت 8000 در حال استفاده است، خطای زیر را می‌بینید:
```
OSError: [WinError 10048] Only one usage of each socket address
```

**راه حل:** 
- سرور قبلی را متوقف کنید (Ctrl+C)
- یا پورت را در `backend/server.py` تغییر دهید

## نکات مهم

1. **همیشه از طریق سرور باز کنید** - نه با دابل کلیک روی فایل
2. **سرور باید در حال اجرا باشد** - قبل از باز کردن مرورگر
3. **از آدرس localhost استفاده کنید** - نه file://

## اگر مشکل حل نشد

1. مطمئن شوید Python نصب است:
   ```bash
   python --version
   ```

2. مطمئن شوید در مسیر درست هستید:
   ```bash
   cd backend
   python server.py
   ```

3. لاگ‌های سرور را بررسی کنید - ممکن است خطایی نمایش داده شود
